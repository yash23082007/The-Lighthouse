# 🚀 Getting Started

## 1. Fork the Repository

Fork the repository to your GitHub account.

## 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/The-Lighthouse.git

cd The-Lighthouse
```

## 3. Add Upstream Remote

```bash
git remote add upstream https://github.com/anushkasark08/The-Lighthouse.git
```

Verify:

```bash
git remote -v
```

# 🌿 Branch Naming Convention

Please avoid committing directly to `main`.

Use:

```text
feature/short-description
fix/issue-name
docs/update-readme
style/navbar-improvements
```

Examples:

```bash
git checkout -b fix/mobile-navbar
git checkout -b style/hero-section-animation
git checkout -b docs/update-contributing
```

---

# 🎨 Coding Guidelines

## HTML

- Use semantic HTML elements.
- Maintain proper indentation.
- Add meaningful alt text for images.
- Avoid unnecessary div nesting.

## CSS

- Use existing CSS variables whenever possible.
- Keep styling consistent with the luxury theme.
- Maintain responsive layouts.
- Avoid duplicate styles.

## JavaScript

- Write clean and readable code.
- Use descriptive variable names.
- Comment complex logic.
- Avoid unnecessary global variables.

# 📱 Responsiveness Guidelines

All UI changes should be tested on:

- Mobile devices
- Tablets
- Desktop screens

The website should remain visually consistent across all screen sizes.

# ♿ Accessibility Guidelines

Contributors are encouraged to:

- Add proper alt text.
- Improve keyboard navigation.
- Maintain sufficient color contrast.
- Use semantic elements.
- Ensure readable font sizes.

Accessibility improvements are always welcome.

# 📸 UI Contribution Guidelines

For any visual changes:

- Include screenshots in your Pull Request.
- Provide before/after comparisons if possible.
- Ensure the premium dark aesthetic is maintained.

# 📝 Commit Message Style

Examples:

```text

style: improve reservation section spacing

```

Keep commit messages:

- Short
- Clear
- Descriptive

# ✅ Testing Your Changes

Before opening a Pull Request:

- Test on Chrome.
- Test on Firefox.
- Test responsive layouts.
- Verify images load correctly.
- Check console for errors.
- Ensure no existing functionality breaks.

---

# 🚀 Pull Request Guidelines

1. Fork the repository.
2. Create a new branch.
3. Make your changes.
4. Commit your changes.
5. Push your branch.

```bash
git push origin your-branch-name
```

6. Open a Pull Request.

Please include:

- What you changed.
- Why you changed it.
- Screenshots for UI changes.
- Related issue number.

Example:

```text
Closes #12
Fixes #7
```

---

# 🐛 Reporting Issues

When opening an issue, include:

- Clear title.
- Steps to reproduce.
- Expected behavior.
- Actual behavior.
- Browser information.
- Screenshots if applicable.
